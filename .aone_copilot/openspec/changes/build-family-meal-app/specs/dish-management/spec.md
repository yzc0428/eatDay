# 功能规范：菜品管理

## 概述
菜品管理模块提供完整的 CRUD 功能，允许用户添加、编辑、删除和查看菜品信息。

## ADDED Requirements

### Requirement: 添加新菜品
系统 SHALL 允许用户创建新的菜品记录，包含名称、分类、价格和图片等信息。

#### Scenario: 用户添加一个新菜品
**Given** 用户在菜品管理页面  
**When** 用户点击"添加菜品"按钮  
**Then** 显示菜品添加表单

**When** 用户填写菜品信息：
- 菜品名称："宫保鸡丁"
- 分类：菜品
- 价格：28.00
- 上传图片（可选）

**And** 用户点击"保存"按钮  
**Then** 系统验证输入信息  
**And** 将菜品保存到 LocalStorage  
**And** 显示成功提示  
**And** 返回菜品列表页面  
**And** 新菜品出现在列表中

#### Scenario: 添加菜品时缺少必填信息
**Given** 用户在菜品添加表单页面  
**When** 用户未填写菜品名称  
**And** 用户点击"保存"按钮  
**Then** 显示错误提示："请输入菜品名称"  
**And** 表单不提交  
**And** 焦点定位到名称输入框

#### Scenario: 添加菜品时上传图片
**Given** 用户在菜品添加表单页面  
**When** 用户点击"上传图片"按钮  
**And** 选择一张图片文件（JPG/PNG，小于 500KB）  
**Then** 系统压缩图片  
**And** 将图片转换为 Base64 格式  
**And** 在表单中显示图片预览  
**And** 用户可以删除或重新上传图片

#### Scenario: 上传的图片超过大小限制
**Given** 用户在菜品添加表单页面  
**When** 用户选择一张大于 500KB 的图片  
**Then** 显示错误提示："图片大小不能超过 500KB，请选择其他图片"  
**And** 图片不被上传

### Requirement: 编辑现有菜品
系统 SHALL 允许用户修改已存在的菜品信息。

#### Scenario: 用户编辑菜品信息
**Given** 用户在菜品列表页面  
**When** 用户点击某个菜品的"编辑"按钮  
**Then** 显示菜品编辑表单  
**And** 表单中预填充该菜品的现有信息

**When** 用户修改菜品价格从 28.00 到 32.00  
**And** 用户点击"保存"按钮  
**Then** 系统更新 LocalStorage 中的菜品信息  
**And** 更新 updatedAt 时间戳  
**And** 显示成功提示  
**And** 返回菜品列表  
**And** 列表中显示更新后的价格

#### Scenario: 编辑时取消操作
**Given** 用户在菜品编辑表单页面  
**When** 用户修改了部分信息  
**And** 用户点击"取消"按钮  
**Then** 显示确认对话框："确定要放弃修改吗？"  
**When** 用户点击"确定"  
**Then** 返回菜品列表  
**And** 菜品信息保持不变

### Requirement: 删除菜品
系统 SHALL 允许用户删除不需要的菜品。

#### Scenario: 用户删除一个菜品
**Given** 用户在菜品列表页面  
**When** 用户点击某个菜品的"删除"按钮  
**Then** 显示确认对话框："确定要删除【宫保鸡丁】吗？此操作不可恢复。"

**When** 用户点击"确定"  
**Then** 系统从 LocalStorage 中删除该菜品  
**And** 从列表中移除该菜品  
**And** 显示成功提示："菜品已删除"

#### Scenario: 删除菜品时取消操作
**Given** 用户在删除确认对话框  
**When** 用户点击"取消"按钮  
**Then** 关闭对话框  
**And** 菜品保持不变

### Requirement: 查看菜品列表
系统 SHALL 允许用户浏览所有已添加的菜品。

#### Scenario: 用户查看菜品列表
**Given** 用户打开应用  
**When** 用户进入菜品管理页面  
**Then** 显示所有菜品的列表  
**And** 每个菜品显示：
  - 菜品图片（如有）
  - 菜品名称
  - 分类标签
  - 价格
  - 收藏状态图标
  - 操作按钮（编辑、删除）

#### Scenario: 菜品列表为空
**Given** 用户首次使用应用  
**When** 用户进入菜品管理页面  
**Then** 显示空状态提示："还没有添加任何菜品"  
**And** 显示"添加第一个菜品"按钮

### Requirement: 搜索菜品
系统 SHALL 允许用户通过名称搜索菜品。

#### Scenario: 用户搜索菜品
**Given** 用户在菜品列表页面  
**And** 列表中有多个菜品  
**When** 用户在搜索框输入"鸡"  
**Then** 列表实时过滤  
**And** 只显示名称包含"鸡"的菜品  
**And** 显示搜索结果数量："找到 3 个菜品"

#### Scenario: 搜索无结果
**Given** 用户在菜品列表页面  
**When** 用户搜索"不存在的菜品"  
**Then** 显示空状态："未找到匹配的菜品"  
**And** 显示"清除搜索"按钮

### Requirement: 菜品收藏
系统 SHALL 允许用户将常用菜品标记为收藏。

#### Scenario: 用户收藏一个菜品
**Given** 用户在菜品列表页面  
**When** 用户点击某个菜品的收藏图标  
**Then** 图标变为实心状态  
**And** 该菜品的 isFavorite 属性设置为 true  
**And** 显示提示："已添加到收藏"

#### Scenario: 用户取消收藏
**Given** 用户在菜品列表页面  
**And** 某个菜品已被收藏  
**When** 用户再次点击该菜品的收藏图标  
**Then** 图标变为空心状态  
**And** 该菜品的 isFavorite 属性设置为 false  
**And** 显示提示："已取消收藏"

## 数据模型

```typescript
interface Dish {
  id: string;                    // UUID
  name: string;                  // 菜品名称（必填，最长 50 字符）
  category: DishCategory;        // 分类（必填）
  price: number;                 // 价格（必填，保留两位小数）
  image?: string;                // Base64 图片（可选）
  isFavorite: boolean;           // 是否收藏（默认 false）
  createdAt: number;             // 创建时间戳
  updatedAt: number;             // 更新时间戳
}

type DishCategory = 'main' | 'dish' | 'soup' | 'drink';
```

## 验证规则

1. **菜品名称**：
   - 必填
   - 长度：1-50 字符
   - 不能只包含空格

2. **分类**：
   - 必选
   - 只能是预定义的四个分类之一

3. **价格**：
   - 必填
   - 必须大于 0
   - 最多两位小数
   - 最大值：9999.99

4. **图片**：
   - 可选
   - 格式：JPG、PNG、WEBP
   - 大小：最大 500KB
   - 自动压缩到合适尺寸

## 性能要求

- 菜品列表加载时间 < 500ms
- 搜索响应时间 < 100ms（使用防抖）
- 图片上传和压缩 < 2s
- 支持至少 200 个菜品的存储和展示
