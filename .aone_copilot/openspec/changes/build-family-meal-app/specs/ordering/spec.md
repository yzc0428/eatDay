# 功能规范：点餐功能

## 概述
点餐模块允许用户选择菜品、设置数量，生成点餐清单并保存到历史记录。

## ADDED Requirements

### Requirement: 选择菜品进行点餐
系统 SHALL 允许用户从菜品列表中选择菜品加入点餐清单。

#### Scenario: 用户添加菜品到点餐清单
**Given** 用户在点餐页面  
**And** 页面显示所有可用菜品  
**When** 用户点击"宫保鸡丁"菜品卡片  
**Then** 该菜品添加到点餐清单  
**And** 默认数量为 1  
**And** 点餐清单显示该菜品  
**And** 显示添加成功的视觉反馈

#### Scenario: 重复添加同一菜品
**Given** 用户在点餐页面  
**And** 点餐清单中已有"宫保鸡丁"（数量 1）  
**When** 用户再次点击"宫保鸡丁"  
**Then** 该菜品数量增加到 2  
**And** 不创建重复条目  
**And** 更新总价

### Requirement: 调整菜品数量
系统 SHALL 允许用户增加或减少点餐清单中菜品的数量。

#### Scenario: 增加菜品数量
**Given** 用户在点餐清单中  
**And** "宫保鸡丁"当前数量为 2  
**When** 用户点击"+"按钮  
**Then** 数量增加到 3  
**And** 该菜品小计更新为 96.00（32 × 3）  
**And** 订单总价更新

#### Scenario: 减少菜品数量
**Given** 用户在点餐清单中  
**And** "宫保鸡丁"当前数量为 3  
**When** 用户点击"-"按钮  
**Then** 数量减少到 2  
**And** 该菜品小计更新  
**And** 订单总价更新

#### Scenario: 数量减少到 0
**Given** 用户在点餐清单中  
**And** "宫保鸡丁"当前数量为 1  
**When** 用户点击"-"按钮  
**Then** 显示确认对话框："确定要移除该菜品吗？"  
**When** 用户点击"确定"  
**Then** 该菜品从点餐清单中移除  
**And** 订单总价更新

#### Scenario: 直接输入数量
**Given** 用户在点餐清单中  
**When** 用户点击数量输入框  
**And** 输入数字 5  
**Then** 数量更新为 5  
**And** 小计和总价更新

#### Scenario: 输入无效数量
**Given** 用户在数量输入框中  
**When** 用户输入 0 或负数  
**Then** 显示错误提示："数量必须大于 0"  
**And** 恢复为之前的有效数量

### Requirement: 从收藏快速添加
系统 SHALL 允许用户从收藏列表快速添加常用菜品。

#### Scenario: 查看收藏菜品
**Given** 用户在点餐页面  
**When** 用户点击"收藏"标签  
**Then** 只显示已收藏的菜品  
**And** 显示收藏数量："收藏 (5)"

#### Scenario: 从收藏添加菜品
**Given** 用户在"收藏"标签下  
**When** 用户点击一个收藏的菜品  
**Then** 该菜品添加到点餐清单  
**And** 与普通添加流程相同

### Requirement: 查看点餐清单
系统 SHALL 允许用户查看当前点餐清单的详细信息。

#### Scenario: 显示点餐清单
**Given** 用户在点餐页面  
**And** 已添加多个菜品  
**Then** 点餐清单显示：
  - 每个菜品的名称、单价、数量、小计
  - 菜品总数量
  - 订单总价
  - "清空"和"提交订单"按钮

#### Scenario: 点餐清单为空
**Given** 用户在点餐页面  
**And** 点餐清单中没有任何菜品  
**Then** 显示空状态："还没有选择菜品"  
**And** "提交订单"按钮禁用

### Requirement: 清空点餐清单
系统 SHALL 允许用户一键清空点餐清单。

#### Scenario: 清空点餐清单
**Given** 用户在点餐页面  
**And** 点餐清单中有多个菜品  
**When** 用户点击"清空"按钮  
**Then** 显示确认对话框："确定要清空点餐清单吗？"  
**When** 用户点击"确定"  
**Then** 清空所有菜品  
**And** 总价重置为 0  
**And** 显示空状态

### Requirement: 提交订单
系统 SHALL 允许用户提交点餐清单，生成历史记录。

#### Scenario: 提交订单
**Given** 用户在点餐页面  
**And** 点餐清单中有菜品  
**When** 用户点击"提交订单"按钮  
**Then** 显示订单确认对话框  
**And** 显示订单详情：
  - 菜品列表
  - 总数量
  - 总价格
  - 当前日期时间

**When** 用户点击"确认提交"  
**Then** 创建订单记录  
**And** 保存到 LocalStorage  
**And** 清空点餐清单  
**And** 显示成功提示："订单已提交"  
**And** 提供"查看历史"按钮

#### Scenario: 快速重新点餐
**Given** 用户刚提交完订单  
**When** 用户在成功提示中点击"继续点餐"  
**Then** 返回点餐页面  
**And** 点餐清单为空  
**And** 可以开始新的点餐

## 计算规则

### 小计计算
```
小计 = 单价 × 数量
保留两位小数
```

### 总价计算
```
总价 = Σ(所有菜品小计)
保留两位小数
```

### 总数量计算
```
总数量 = Σ(所有菜品数量)
```

## UI 规范

### 点餐清单布局
- 位置：页面右侧（桌面端）或底部抽屉（移动端）
- 宽度：320px（桌面端）
- 最大高度：80vh
- 可滚动内容区域

### 菜品卡片（点餐模式）
- 显示菜品图片、名称、价格
- 点击整个卡片添加到清单
- 已添加的菜品显示"已选"标记
- 悬停效果：轻微放大和阴影

### 数量调整控件
- 布局：[-] [数量] [+]
- 按钮大小：32px × 32px
- 数量输入框：60px 宽
- 最大数量：99

## 性能要求
- 添加菜品响应时间 < 100ms
- 数量调整响应时间 < 50ms
- 价格计算准确无误
- 支持至少 50 个菜品同时在清单中
